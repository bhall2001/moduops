# Welcome to ModuOps #
Software designed around the nuances of operating a modular Free-mo layout. ModuOps takes inspiration from a software packaged called "Mad River and Big Timber" developed for modular operations in the late 1990s and early 2000s.

ModuOps is in it's infancy (at best) but it is starting to take shape. The software is written in LiveCode and is licensed under GNU GPL v3. If you have any experience in LiveCode programming we can use your help.

## What is ModuOps ##
ModuOps is a Trainlist generator for modular (Free-mo specifically) and home layouts. The software takes into account the issues related to a modular layout that changes at every setup and rollingstock that is not known in advance. My hope is that setting up the software, adding industries, rollingstock and trains is a easy as can be so that Trainlists can be generated in short order during layout operating sessions.

## Development Test ##
ModuOps is projected to be used for the NE Free-mo Operating Sessions at the Amherst Railway Society Model Train Show in West Springfield, MA on January 24th and 25th 2015. If you'd like to see the software stop by the NE Free-mo layout and have a look.

## Status of App ##
ModuOps is under active development. At this point, I am using the software to generate Trainlists on my home layout. All the basic data structures and major features for the 1.0 version are set.

If you choose to download this version be aware there is little documentation right now (I'll get there). However, in general, the software is usable and does not crash (at least for me). If you poke around a bit I suspect you'll figure out how it works and be able to try it out.
